1) What the Firefox Health Report Measures

The Firefox Health Report (FHR) has 4 categories of metrics:

Configuration
Device hardware, operating system, Firefox version
Customizations
Add-ons, count and type
Performance
Timing of browser events, rendering, session restores
Wear and Tear
Length of session, how old a profile is, count of crashes, count of pages

2) Detailed Description

2.1) Configuration

Device hardware, operating system, Firefox version

Metric Description
appABI Binary interface for XPCOM (https://developer.mozilla.org/XPCOM_ABI)
appBuildID Identifier for this build of the application based on the date the build was started
appBuildTargetOS Operating System for which this build was configured
appDistribution Identifier for the particular distribution of the application in use. It is taken from the distribution.id preference
appDistributionVersion Version of the distribution, taken from the distribution.version preference
appHotFixVersion The version number of any Mozilla hotfix applied to the application
appID Identifier unique to the application that indicates it is Firefox or Thunderbird, etc. Same for all installations of an application regardless of the value of appName
appName The configured name of the application (Firefox/Aurora/Nightly for instance)
appUpdateChannel The channel this installation checks for application updates (nightly/aurora/beta/release)
appVendor The name of the company/provider who built this version
appVersion The current version of the product
cpuCount Number of processor cores
isDefaultBrowser Whether the application is currently configured to be the default browser for the operating system
locale Locale currently being used
memSize Amount of RAM in megabytes
osName Currently running Operating System (kernel) name
osVersion Currently running Operating System (kernel) version
platformBuildID The build identifier of the Gecko platform code
platformVersion The version of the Gecko platform code
processorArch Processor architecture
version.date The date this version was first used with this installation profile
version.version An application version which has been used with this installation profile

2.2) Customizations

Add-ons, count and type

Metric Description
addonCounts The number of add-ons of this type that existed in the profile on this date
appDisabled True if the application automatically disabled the add-on
foreignInstall Whether this add-on was installed by the user or automatically by another application
hasBinaryComponents Whether this add-on has native code.
id The identifier of this add-on as defined by the add-on developer
installDate The most recent date this add-on was installed (previous information about an uninstalled add-on is not kept).
search The number of searches performed with this search engine from this part of the UI
scope Whether this add-on was installed in the application directory or profile directory
type Whether this add-on is an extension or a plug-in
updateDate The most recent date this add-on was updated (previous information about an uninstalled add-on is not kept).
userDisabled True if the user manually disabled the add-on
version The current version of this add-on

2.3) Performance

Timing of browser events, rendering, session restores

Metric Description
firstPaint How long in milliseconds it took during startup for the application to render the UI and display the window
main How long in milliseconds it took during startup for the application to load libraries and finish basic initialization
sessionRestored How long in milliseconds it took during startup for the application to restore tabs from the previous session

2.4) Wear and Tear

Length of session, how old a profile is, count of crashes, count of pages

Metric Description
aborted The number of times the application was run and not cleanly shut down.
abortedActiveTime The time in seconds that the aborted sessions were not idle
abortedTime The time in seconds that the aborted sessions were running
appProfileAge The number of days since this application profile directory was created
completed The number of times the application was run and cleanly shut down.
completedActiveTime The time in seconds that the completed sessions were not idle
completedTime The time in seconds that the completed sessions were running
crashCountPending Total number of crashes that were recorded but not yet submitted
crashCountSubmitted Total number of crashes that have been submitted
currentSessionActiveTime Time in seconds the app has not been idle
currentSessionTime Time in seconds since the app was last started
placesBookmarksCount Total number of bookmarks
placesPageCount Total number of pages listed in the history
uptime How long in minutes the current session has been running
